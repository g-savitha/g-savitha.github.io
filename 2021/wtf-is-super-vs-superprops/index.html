WTF is super() vs super(props)

29 January, 2021  ·   React

TL;DR 🚀

If you want to use this.props inside constructor of derived class, use super(props).

While checking react code, you might have come across a line super(props), sometimes super(). Ever wondered what’s the difference between two? Why do we need to pass props ? Do we always need to pass props ?

Consider the example below.

 1import React, { Component } from 'react';
 2class Game extends Component {
 3    constructor(props) {
 4      super(props);
 5        this.state = {
 6          player : "Robin",
 7          score : 25
 8        }
 9    }
10    render() {
11        return (
12            <div>
13                <h1>Cricket</h1>
14                <p>Current Player :{this.state.player}</p>
15                <p>Current Score : {this.state.score}</p>
16            </div>
17        );
18    }
19}
20export default Game;

Every time we are writing a component, we are extending it from React component (The base component class) and that’s really important if we don’t do that we would be missing a ton of react functionality.

Let’s step out of react world for just a moment and let’s mimic our own component class

 1class Component {
 2    constructor() {
 3        console.log("INSIDE COMPONENT CONSTRUCTOR")
 4    }
 5}
 6class Game extends Component {
 7    constructor(){
 8        console.log("INSIDE GAME CONSTRUCTOR")
 9    }
10}

Lets make a new game

1let cricket = new Game();

So you might think, when we initialized a new game, a constructor is called and that should be printed out, unfortunately we get an error says : Must call super constructor before accessing ‘this’ . Loosely translated, call a super constructor in derived class before basically doing anything.

Problem is we are extending the base class but we never actually called its constructor and that’s super important to do in the derived class’s constructor (Inside Game’s constructor here), we need to make sure that we’re calling the class (it’s constructor) we are extending, especially in React because that’s how we call all the magic setup stuff that React does and takes care of out of the box for us. so we call super()

1class Game extends Component {
2    constructor(){
3      super()
4        console.log("INSIDE GAME CONSTRUCTOR")
5    }
6}

Now try to instantiate again.

1let cricket = new Game();
2//output:
3  //INSIDE COMPONENT CONSTRUCTOR
4  //INSIDE GAME CONSTRUCTOR

What does super(props) do then?

Lets look at this example:

 1import React, { Component } from 'react';
 2
 3export default class Food extends Component {
 4    render() {
 5        return (
 6            <div>
 7                <h1>HI I AM FOOD COMPONENT</h1>
 8            </div>
 9        );
10    }
11}

 1import React, { Component } from 'react'
 2import Food from "./Food"
 3class App extends Component {
 4  render(){
 5    return (
 6    <div>
 7      <Food item = "PineApple" nutrition="10"/>
 8    </div>);
 9  }
10}
11
12export default App;

We are passing two props to Food component. You can think of it like we are creating two objects for Food class new Food({item : "pineapple", nutrition: "10"})

 1import React, { Component } from 'react';
 2
 3export default class Food extends Component {
 4    constructor() {
 5        super();
 6        console.log(this.props); // undefined.
 7    }
 8    render() {
 9        return (
10            <div>
11                <h1>HI I AM FOOD COMPONENT</h1>
12            </div>
13        );
14    }
15}

We get output as Undefined even though we have props. OK, what’s happening here is if we want to access this.props in the constructor, we have to pass props in super (super(props)). This is kind of bizzare, lets consider this example.

 1import React, { Component } from 'react';
 2
 3export default class Food extends Component {
 4    constructor() {
 5        super();
 6        console.log(this.props); // undefined.
 7        console.log(props) // {item:"pineapple", nutrition:10}
 8    }
 9    render() {
10        return (
11            <div>
12                <h1>{this.props.item}</h1> /*Prints out pineapple*/
13            </div>
14        );
15    }
16}

However if you do

 1import React, { Component } from 'react';
 2
 3export default class Food extends Component {
 4    constructor() {
 5        super(props);
 6        console.log(this.props); // {item:"pineapple", nutrition:10}
 7    }
 8    render() {
 9        return (
10            <div>
11                <h1>{this.props.item}</h1> /*Prints out pineapple*/
12            </div>
13        );
14    }
15}

To conclude, If you want to use this.props inside constructor, you need to pass props in super(), otherwise it’s okay to not pass props to super() as we see that irrespective of passing it to super(), this.props is available inside render() function.

Hopefully that was not convoluted and crazy, until next time. Happy Coding! 🎉 💻
